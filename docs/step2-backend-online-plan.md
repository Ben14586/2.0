# Step 2 Backend Online Plan

## Objective

Make the Netlify storefront fully online by connecting it to a real hosted backend. After Step 2, real customer orders should create durable `ORD-...` IDs and appear in the admin panel.

## Current Status

- Netlify static frontend is already online.
- Local backend package is ready: `backend-deploy-latest.zip`.
- Step 2 backend release folder is generated by:

```powershell
npm run step2:prepare
```

- External blocker: Render/Railway account session or deploy token is required to upload/start the backend.

## Upload Target

Recommended first target: Render.

Why:

- `render.yaml` is already present.
- `/health` route exists.
- Persistent disk config is prepared.
- Python start command is simple: `python server.py`.

Railway is acceptable if Render is not available.

## Backend Upload Source

Use one of these:

- `backend-deploy-latest.zip`
- extracted folder: `step2-backend-release/`

The extracted folder contains:

- backend source files
- Render/Railway/Docker/Procfile config
- `uploads/game-images/`
- `STEP2_UPLOAD_README.md`
- `RENDER_ENV_PLACEHOLDERS.txt`
- `step2-checksums.json`

It intentionally excludes:

- `database.db`
- slip uploads
- local `.env`

## Render Settings

- Runtime: Python
- Build command: `pip install -r requirements.txt`
- Start command: `python server.py`
- Health check path: `/health`
- Persistent disk:
  - Mount path: `/var/data`
  - Size: 1GB or higher

## Required Host Secrets

Set these in Render/Railway dashboard, not in public files:

```text
HOST=0.0.0.0
DATABASE_FILE=/var/data/database.db
UPLOADS_DIR=/var/data/uploads
ADMIN_USERNAME=admin
ADMIN_BOOTSTRAP_PASSWORD=<strong-private-password>
ADMIN_SESSION_HOURS=8
COOKIE_SECURE=1
COOKIE_SAMESITE=None
PUBLIC_SITE_URL=https://store-game-0.netlify.app
ADMIN_SITE_URL=https://store-game-0.netlify.app
PUBLIC_API_BASE_URL=https://your-backend-host.example.com
ALLOWED_ORIGINS=https://store-game-0.netlify.app
ALLOW_FILE_ORIGIN=0
```

Optional:

```text
ADMIN_KEY=<private-emergency-key-or-blank>
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
LINE_NOTIFY_TOKEN=
```

## After Backend URL Exists

Run:

```powershell
npm run go-live:connect -- https://your-backend-host.example.com
```

This will:

- verify backend `/health`
- write `PUBLIC_API_BASE_URL`
- rebuild `netlify-deploy-latest.zip`
- run QA
- run online smoke test
- run Phase 4 dry run
- update `docs/production-online-status.md`

## Netlify Update

After `go-live:connect` passes:

1. Upload the refreshed `netlify-deploy-latest.zip` to Netlify.
2. Open `https://store-game-0.netlify.app`.
3. Create one real browser order.

## Acceptance Criteria

- Backend `/health` returns `success: true`.
- Netlify site creates order IDs starting with `ORD-`, not `WEB-`.
- Admin sees the new order.
- Admin can change order status.
- Public tracker can find the order.
- Admin backup download works.

## Next Upgrade After Step 2

After the first successful `ORD-...` order:

- tighten production admin password policy
- decide whether `ADMIN_KEY` should remain enabled or blank
- test one valid slip image and one invalid file
- export Excel after first real test order
- take one admin backup before promotion
